Description

We developed ROQY in a team of six developers and four designers. It was a module from the computer science studies where the company Liveperson gave us the task to build a web-based chatbot-configurator.

Big Companies with many customers get a lot of recurring requests. With a chatbot you can automate these inquiries. Customers get a quick answer because they don't have to wait until an employee answers them and the customer agents have more time for important tasks. (Or get fired because ROQY does everything, but unemployment because of automatisation is another topic we can discuss about for hours next time!)

In the last few years Conversational UIs became a big topic in the design world. Some argue that they are the future of the web interface. With the big tech companies jumping onto this trend this might be true. And there are reasons why. CUIs are easy to develop and work cross-platfrom. They are easy to understand because they work just the way we talk and are barrier-free because there is just text and no interface you have to translate. With these advantages I think chatbots came to stay.

Chatbot Configuration

Here is the main part of ROQY. You can see the configuration environment where you create the bot conversation. We put every snippet of a conversation (being a question and an answer) into a block. On the left we defined blocks which don't need to be in a conversation structure but need to be defined with every bot. These blocks are a welcome message and a block for the event of the bot not understanding what the enduser writes.

The rest of the conversation is structured into tabs like the folder system of the mac. This is for clarity of the configuration environment and to guide the end user through a conversation.

With the panel on the right you type the text into the blocks. Here you define wordchunks we call blobs for the input the enduser gives us and with language processing we check if the input is similar to one of our blobs. When this is true the bot gives out the corresponding answer.

Testing

You can test your bot to check on mistakes in the conversation flow. You can switch between bot configuration and testing anytime. When you are satisfied with it, you can deploy the bot.
